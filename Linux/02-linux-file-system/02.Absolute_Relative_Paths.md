# Linux Paths and Tree Command — Professional Guide

---

# Introduction

Understanding paths in Linux is one of the most important skills for working efficiently in the terminal.

This guide explains:

- Absolute Path
- Relative Path
- `tree` command
- `tree -d`
- `tree -f`

The concepts are explained using practical examples and easy memory techniques.

---

# 1. Absolute Path

## Definition

An **absolute path** is the complete path to a file or directory starting from the root directory `/`.

It always begins with:

```bash
/
```

---

## Structure

```bash
/parent_directory/subdirectory/file
```

---

## Examples

```bash
/home/mayur/Documents/file.txt
```

```bash
/etc/passwd
```

```bash
/var/log/messages
```

---

## Key Characteristics

- Starts from the root directory `/`
- Independent of the current working directory
- Always gives the exact location

---

## Memory Tip

> Absolute path = Full address of a house.

Example:

```text
Country → City → Area → Street → House Number
```

Similarly:

```bash
/home/mayur/Documents/file.txt
```

---

# 2. Relative Path

## Definition

A **relative path** is the path relative to the current working directory.

It does not start with `/`.

---

## Structure

```bash
directory/file
```

or

```bash
../directory/file
```

---

## Examples

Assume current directory:

```bash
/home/mayur
```

Then:

```bash
Documents/file.txt
```

means:

```bash
/home/mayur/Documents/file.txt
```

---

## Common Symbols

| Symbol | Meaning           |
| ------ | ----------------- |
| `.`    | Current directory |
| `..`   | Parent directory  |
| `~`    | Home directory    |

---

## Examples

### Current Directory

```bash
./script.sh
```

---

### Parent Directory

```bash
../notes.txt
```

---

### Home Directory

```bash
~/Downloads
```

---

## Key Characteristics

- Depends on the current location
- Shorter and easier to type
- Commonly used in daily terminal work

---

## Memory Tip

> Relative path = Directions from your current location.

Example:

```text
Go one floor up → enter another room
```

Equivalent Linux path:

```bash
../another_directory
```

---

# Absolute Path vs Relative Path

| Feature                      | Absolute Path | Relative Path |
| ---------------------------- | ------------- | ------------- |
| Starts with `/`              | Yes           | No            |
| Depends on current directory | No            | Yes           |
| Complete location            | Yes           | No            |
| Easier for scripting         | Yes           | Sometimes     |
| Shorter to type              | No            | Yes           |

---

# Examples Comparison

Assume current directory:

```bash
/home/mayur
```

| Purpose             | Absolute Path                 | Relative Path     |
| ------------------- | ----------------------------- | ----------------- |
| Open file           | `/home/mayur/file.txt`        | `file.txt`        |
| Open Documents file | `/home/mayur/Documents/a.txt` | `Documents/a.txt` |
| Access parent file  | `/home/notes.txt`             | `../notes.txt`    |

# 3. Tree Command

## Definition

The `tree` command displays directories and files in a tree-like structure.

It helps visualize the filesystem hierarchy.

---

# Basic Syntax

```bash
tree
```

---

# Example

```bash
tree
```

Output:

```text
.
├── Documents
│   ├── notes.txt
│   └── project
├── Downloads
└── Pictures
```

---

# Benefits of Tree Command

- Visualizes directory structure
- Easy to understand nested folders
- Useful for documentation
- Helpful in projects and system administration

---

# Installing Tree Command

Some Linux distributions may not have `tree` installed.

## Ubuntu/Debian

```bash
sudo apt install tree
```

---

## RHEL/CentOS

```bash
sudo yum install tree
```

---

# 4. tree -d

## Purpose

Displays only directories.

Files are hidden.

---

## Syntax

```bash
tree -d
```

---

## Example

```bash
tree -d
```

Output:

```text
.
├── Documents
│   └── project
├── Downloads
└── Pictures
```

---

## Use Cases

- Viewing folder hierarchy
- Understanding project structure
- Exploring system directories

---

## Memory Tip

> `-d` = directories only

---

# 5. tree -f

## Purpose

Displays the full path prefix for each file and directory.

---

## Syntax

```bash
tree -f
```

---

## Example

```bash
tree -f
```

Output:

```text
.
├── ./Documents
│   ├── ./Documents/notes.txt
│   └── ./Documents/project
├── ./Downloads
└── ./Pictures
```

---

## Benefits

- Shows complete file paths
- Useful in scripts
- Helpful for debugging and documentation

---

## Memory Tip

> `-f` = full path

---

# Combining Options

## Display only directories with full paths

```bash
tree -df
```

---

# Practical Examples

---

## View Entire Home Directory Structure

```bash
tree /home/mayur
```

---

## View Only Directories

```bash
tree -d /etc
```

---

## View Full Paths

```bash
tree -f /var/log
```

---
