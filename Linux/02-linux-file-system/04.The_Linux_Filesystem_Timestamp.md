# Introduction

Every file in Linux stores timestamp information.

These timestamps help administrators and users track:

- file access,
- content modifications,
- and metadata changes.

Linux maintains three primary timestamps for every file:

1. Access Time (`atime`)
2. Modification Time (`mtime`)
3. Change Time (`ctime`)

Understanding these timestamps is important for:

- system administration,
- troubleshooting,
- backups,
- auditing,
- and security analysis.

# 1. Access Time (`atime`)

## Definition

`atime` records the last time a file was accessed or read.

Examples of access:

- opening a file,
- reading file contents,
- using `cat`,
- viewing a document.

---

## View Access Time Using `ls`

```bash
ls -lu file.txt
```

---

## Example

```bash
cat file.txt
ls -lu file.txt
```

---

## Important Notes

- Reading a file updates `atime`
- Simply viewing file contents may change it
- Some systems disable frequent `atime` updates for performance reasons

---

# 2. Modification Time (`mtime`)

## Definition

`mtime` records the last time the file content was modified.

Examples:

- editing text,
- adding data,
- deleting content.

---

## View Modification Time Using `ls`

```bash
ls -l file.txt
```

or

```bash
ls -lt file.txt
```

---

## Example

```bash
echo "Linux" >> file.txt
ls -l file.txt
```

---

## Important Notes

- Changing file contents updates `mtime`
- Most commonly used timestamp
- Important for backups and synchronization tools

---

# 3. Change Time (`ctime`)

## Definition

`ctime` records the last time file metadata changed.

Metadata includes:

- permissions,
- ownership,
- inode information,
- links.

---

## View Change Time Using `ls`

```bash
ls -lc file.txt
```

---

## Example

```bash
chmod 755 file.txt
ls -lc file.txt
```

---

## Important Notes

- `ctime` changes when metadata changes
- Editing file permissions updates `ctime`
- `ctime` is not "creation time"

---

# Understanding the Difference

| Timestamp | Meaning           | Updated When         |
| --------- | ----------------- | -------------------- |
| `atime`   | Access time       | File is read         |
| `mtime`   | Modification time | File content changes |
| `ctime`   | Change time       | Metadata changes     |

# Using the `stat` Command

The `stat` command displays detailed file information including all timestamps.

---

# Basic Syntax

```bash
stat filename
```

---

# Example

```bash
stat file.txt
```

---

# Sample Output

```text
File: file.txt
Size: 120         Blocks: 8
Access: 2026-05-11 10:20:30
Modify: 2026-05-11 09:50:12
Change: 2026-05-11 09:55:40
```

---

# Understanding `stat` Output

| Field  | Meaning                                  |
| ------ | ---------------------------------------- |
| Access | Last file access time (`atime`)          |
| Modify | Last content modification time (`mtime`) |
| Change | Last metadata change time (`ctime`)      |

| Command  | Purpose                   |
| -------- | ------------------------- |

| `ls -lt` | Sort by modification time |

# Important Concepts

---

## `ctime` Is Not Creation Time

Many beginners confuse:

```text
ctime = creation time
```

This is incorrect.

`ctime` means:

- inode change time,
- metadata change time.

---

## Linux Usually Does Not Store Creation Time

Traditional Linux filesystems generally do not store:

- true file creation timestamps.

Modern filesystems may support it separately.
