# Linux File System — Quick Memory Guide

> “Do not memorize Linux folders blindly. Understand their purpose.”

---

# Root Directory

```bash
/
```

The `/` directory is called the **root directory**.

It is the starting point of the entire Linux filesystem.

Everything in Linux exists under `/`.

---

# Core Idea to Remember

The easiest way to remember Linux folders permanently is:

- Understand the purpose
- Associate the folder with a real-world role
- Use them repeatedly

---

# Important Linux Directories

---

## `/boot`

Contains files required to start the operating system.

Includes:

- Kernel files
- Bootloader files

Memory Tip:

> Linux cannot start without `/boot`.

---

## `/bin`

Contains essential commands available to all users.

Examples:

```bash
ls
cp
mv
cat
```

Memory Tip:

> `bin` stands for binary commands.

---

## `/sbin`

Contains administrative and system management commands.

Usually used by the root user.

Examples:

```bash
reboot
fdisk
shutdown
```

Memory Tip:

> `s` means system or superuser tools.

---

## `/etc`

Contains system-wide configuration files.

Examples:

- Network configuration
- User authentication settings
- Service configurations

Memory Tip:

> Think of `/etc` as the system settings directory.

---

## `/home`

Contains personal directories for normal users.

Example:

```bash
/home/mayur
```

Memory Tip:

> Personal workspace for users.

---

## `/root`

Home directory of the root user.

Memory Tip:

> Separate administrator workspace.

---

## `/dev`

Contains device files.

Examples:

- Hard disks
- USB devices
- Terminal devices

Memory Tip:

> Linux treats hardware devices as files.

---

## `/proc`

Virtual filesystem containing live system and process information.

Examples:

- CPU details
- Memory usage
- Running process information

Memory Tip:

> Real-time system monitoring area.

---

## `/sys`

Contains kernel and hardware-related information.

Memory Tip:

> Interface between Linux kernel and hardware.

---

## `/usr`

Contains most installed applications, utilities, and documentation.

Includes:

- User applications
- Shared libraries
- Manuals

Memory Tip:

> Main software storage area.

---

## `/var`

Contains files that frequently change.

Examples:

- Logs
- Cache
- Mail
- Databases

Memory Tip:

> `var` means variable data.

---

## `/tmp`

Contains temporary files.

Used by:

- Applications
- Users
- System processes

Memory Tip:

> Temporary workspace cleaned regularly.

# Memory Story

Think about Linux startup in sequence:

1. `/boot` starts the system
2. `/bin` provides essential tools
3. `/etc` loads configurations
4. `/dev` detects hardware
5. `/proc` tracks running processes
6. `/home` opens user workspace
7. `/usr` provides applications
8. `/var` stores logs and changing data
9. `/tmp` manages temporary files
